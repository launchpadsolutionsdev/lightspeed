Heartbeat Gets Sub-Pages, Shopify Intelligence Fixes & More

Heartbeat now splits into two focused sub-pages — Live Dashboard for real-time raffle monitoring and Shopify Intelligence for your store’s daily snapshot — with significant data accuracy fixes across the board.

Since launching Heartbeat last week, we’ve been watching how teams use it during live events. The biggest piece of feedback: the page was doing too much. Real-time raffle velocity and a Shopify store snapshot are both valuable, but they serve different needs at different moments. Today’s update addresses that — and fixes several data accuracy issues we identified along the way.

Two Dedicated Sub-Pages

Heartbeat now has a collapsible sub-menu in the sidebar, just like the Response Assistant. Click the arrow to expand it and you’ll see two focused pages:

  • Live Dashboard — Everything related to your active raffle: sales velocity ticker, vital signs, surge detection, payment breakdown, package performance, hourly heatmaps, jackpot comparisons, and the live orders feed. Polls every 5 seconds for real-time data.
  • Shopify Intelligence — Your Shopify store’s daily snapshot: revenue, orders, average order value, refunds, geographic hotspots, top products, top buyers, and discount code tracking. Auto-refreshes every 2 minutes.

Each sub-page only runs the polling it needs. When you’re on the Live Dashboard, Shopify queries don’t fire. When you switch to Shopify Intelligence, the 5-second raffle ticker stops. This means less unnecessary network traffic and a snappier experience.

Product Revenue Was Showing $0 — Fixed

The “Top Products Today” section was showing product names and unit counts correctly, but every product displayed $0 in revenue. The root cause: when the primary analytics tables are still warming up, the system falls back to parsing line item data from recent orders. That data stores prices in a field called price_cents, but the fallback parser was looking for a field called price — which doesn’t exist. The result: undefined became 0 for every product.

The fix reads price_cents first, with a fallback to price for safety. Product revenue now displays correctly from the first order onward.

Geographic Data Was Showing “Unknown” — Fixed

The “Where Orders Are Coming From” section was displaying “Unknown, Unknown” for every order. This happened because the system only looked at shipping addresses when syncing order data from Shopify — but raffle tickets and digital products don’t have shipping addresses.

We made two changes:

  • The Shopify sync now falls back to billing addresses when no shipping address is available, which captures the vast majority of digital-product orders.
  • For orders that genuinely have no address data at all, the frontend now shows “No Address on File” instead of the confusing “Unknown, Unknown” label.

Live Timestamps So You Know Data Is Fresh

Previously, the Shopify Intelligence page showed today’s date but nothing to indicate when the data was last fetched. If you left the page open for 10 minutes, there was no way to tell whether the numbers were current or stale.

The API now returns a generated_at timestamp with every response. The page header displays “Updated 3:45 PM ET” (or whatever the current time is), and the footer confirms the refresh cadence: “Data resets at midnight ET · Auto-refreshes every 2 minutes.”

Clearer Labeling

The section header previously read “24h Snapshot,” which implied a trailing 24-hour window. In reality, the KPIs, products, and geographic data all use calendar-day queries that reset at midnight Eastern. We renamed it to “Today’s Snapshot” so the numbers match expectations — they represent today so far, and they’ll climb throughout the day.

Shopify Analytics Consolidated

The standalone Shopify Analytics page that previously lived in the sidebar has been retired. All Shopify store data now lives under Heartbeat › Shopify Intelligence, giving you a single place to check your store’s daily performance alongside your raffle data.

Keyboard & Command Palette

Both new sub-pages are accessible from the command palette (Cmd+K or Ctrl+K). Type “heartbeat” or “shopify” to jump directly to either page. Direct URLs also work: /heartbeat/live and /heartbeat/shopify.
