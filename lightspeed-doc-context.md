## Part 1: Product Overview

### What is Lightspeed?

Lightspeed by Launchpad Solutions is an AI-powered productivity suite for hospital lotteries and charitable gaming organizations. From the README:

> "AI-powered productivity suite for hospital lotteries and charitable gaming organizations. Helps staff respond to customer inquiries, analyze data, draft content, and normalize lists — faster and more consistently."

### What Problem Does It Solve?

Lightspeed enables non-technical staff at hospital foundations and charitable gaming organizations to leverage AI (Claude by Anthropic) for:
- **Customer service responses** — Draft professional replies to player inquiries 7x faster
- **Content creation** — Generate emails, social posts, media releases, and ads
- **Data analysis** — Upload Excel/CSV data for automated insights and visualizations
- **List management** — Clean, deduplicate, and normalize customer/seller data
- **Regulatory compliance** — Get jurisdiction-specific guidance on lottery regulations
- **Team coordination** — Bulletin board, content calendar, and shared knowledge base

### Who is the Target User?

Staff at hospital foundations, charitable gaming organizations, and nonprofit lotteries in Canada who manage:
- Customer communications (email, Facebook, phone inquiries)
- Marketing campaigns (draw announcements, winner stories, social media)
- Sales data analysis (BUMP Raffle data, Shopify store analytics)
- Regulatory compliance (AGCO rules, provincial gaming regulations)
- Team coordination (internal announcements, content scheduling)

### Core Tools

| Tool | Function |
|------|----------|
| **Response Assistant** | Paste customer inquiries, get AI-generated responses with tone/length controls |
| **Draft Assistant** | Generate social posts, emails, ads, media releases, and general content |
| **Insights Engine** | Upload Excel/CSV data for automated analysis and visualizations |
| **Data Agent** | Clean and deduplicate customer/seller lists |
| **Ask Lightspeed** | General-purpose AI assistant with tool use (agentic) |
| **Compliance Assistant** | Jurisdiction-specific regulatory guidance chat |
| **Rules of Play Generator** | Generate raffle/lottery rules documents |
| **Heartbeat** | Real-time raffle sales velocity dashboard + Shopify Intelligence |
| **Home Base** | Team bulletin board with posts, comments, reactions |
| **Runway** | Content calendar with events, reminders, and recurrence |

### Multi-Tenant Platform Features
- Each organization gets isolated data, knowledge base, and settings
- Team management with roles (owner / admin / member)
- Knowledge base with pre-built Ontario compliance entries + custom entries + document import
- Analytics dashboard tracking response volumes, categories, ratings, team leaderboard
- Agent instructions for contextual AI guidance
- Shopify integration for store data and analytics
- Billing with Stripe integration, 14-day free trial, monthly/annual plans

## Part 2: Feature Inventory

### 2.1 Response Assistant

- **Feature name:** Response Assistant
- **Location:** Sidebar > Response Assistant (icon: ⚡)
- **Purpose:** Generate professional responses to customer inquiries
- **Sub-pages:** Generator, Thread, Quick Replies, Analytics

#### Generator
- **Inputs:**
  - Customer Inquiry textarea — placeholder: "Paste the customer's email or inquiry here..."
  - Inquiry Type toggle: "📧 Email" (default) or "💬 Facebook"
  - Smart suggestion chips: "Can't find tickets", "Subscription issue", "Location blocked", "Refund request", "Winner question"
  - Agent Instructions textarea (optional) with quick chips: "I've already resolved this issue", "I've already cancelled their subscription", "Refund has been approved and will be visible on your statement in 2-3 business days", "Escalate to manager — don't promise immediate resolution", "I have resent the tickets to your email"
  - Response Options (expandable accordion): Your Name field, Tone slider (Formal to Friendly), Length slider (Brief to Detailed), Checkboxes: "Include helpful links" (checked by default), "Include step-by-step instructions"
  - Language selector: English, French, Spanish
- **Buttons:** "⚡ Generate Response" or "⚡ Generate Facebook Reply" (dynamic based on inquiry type)
- **Outputs:**
  - AI-generated response text (streamed via SSE)
  - Knowledge Base entries cited
  - Response metadata: generated by, time, word count, sentiment badge (😠 Angry, 😤 Frustrated, 😊 Positive, 😐 Neutral)
- **Actions:** Copy Response, Save to Favorites, Save as Template, View Response History, Generate Another
- **Context automatically applied:** Brand voice, org KB, response rules, Shopify order history, calendar events, voice fingerprint
- **File paths:** `frontend/app.js` (lines ~1349+), `backend/src/routes/tools.js`, `backend/src/services/systemPromptBuilder.js`

#### Thread
- **Purpose:** Multi-turn conversation continuation with customer context
- **Features:** Continue conversations, view full thread history, copy thread, export thread

#### Quick Replies (Templates)
- **Purpose:** Browse, save, and manage reusable response templates
- **Features:** Import from library, create custom templates, edit/delete, copy to clipboard

#### Analytics
- **Metrics:** Response generation count, team leaderboard, inquiry category breakdown (chart), monthly response trend, sentiment analysis, top response categories
- **Export:** CSV export of analytics data

### 2.4 Data Agent

- **Feature name:** Data Agent
- **Location:** Sidebar > Data Agent (icon: 🤖)
- **Purpose:** Clean, normalize, and compare lists
- **Modes:**
  1. **List Normalizer** — Upload CSV/Excel, auto-detect format, clean and standardize data, remove duplicates, download normalized file

  4. **List Comparer** — Upload List A and List B, view: only in A / in both / only in B, download each category
  5. **Raw Normalizer** — Upload file, enter custom normalization instructions, AI applies logic, download results
- **Common actions:** Upload File, Download Results, Review Changes, Clear & Start Over

## Part 3: User Workflows

### 3.1 Onboarding / Signup Flow

**Authentication Options:** Google OAuth and Microsoft OAuth (MSAL) — no email/password registration.

**First-Time User Flow:**
1. User logs in via Google or Microsoft
2. System checks if user exists; if not, creates new user with verified email
3. System auto-joins new users to organizations if they have pending invitations
4. Frontend receives `isNewUser: true` and `needsOrganization: true` if user has no organization
5. User sees **Organization Setup Wizard** (multi-step onboarding)

**Organization Setup Wizard Steps:**
1. **Organization Info** — Name, website, license number
2. **Extended Details** — Mission, support email, store location, CEO info, media contact
3. **Brand/Draw Info** — Brand terms, draw time, CTA website, social required line

**Auto-Seeded Content on New Organization:**
- 2 starter response rules:
  - "Never tell the customer to feel free to reach out..."
  - "Start every email response with 'Hi there,' followed by 'Thank you for reaching out.'"
- All system-level content templates copied (email, social, media release, ad templates)
- Organization set to trial status (14-day free trial)

### 3.2 First Login Experience

- Greeting customized by time of day (morning/afternoon/evening variants + 35 "anytime" greetings)
- Example: "Good morning, [FirstName]." or "Tickets won't sell themselves"
- Subscription status check displayed
- "Tips & Tricks" card with 9 rotating suggestions
- Quick access tiles for each tool with descriptions

### 3.3 Main Day-to-Day Workflows

**Workflow 1: Responding to Customer Inquiries**
1. Open Response Assistant
2. Paste customer email/message
3. Select inquiry type (Email or Facebook)
4. Optionally add agent instructions ("Refund approved", etc.)
5. Adjust tone/length if needed
6. Click "Generate Response"
7. Review streamed response
8. Copy to clipboard and paste into email/Facebook
9. Optionally save as favorite or template

**Workflow 2: Creating Marketing Content**
1. Open Draft Assistant
2. Select content type (Email, Social, Media Release, Ad, Write Anything)
3. Enter prompt/instructions
4. Choose tone and variant count
5. Generate content
6. Copy (plain text, HTML, Mailchimp format, or Meta Ads Manager JSON)

**Workflow 3: Analyzing Sales Data**
1. Open Insights Engine
2. Select report type (Customer Purchases, Customers, Payment Tickets, Sellers)
3. Upload CSV/Excel from BUMP Raffle
4. View auto-generated charts and smart insights
5. Export as JSON, CSV, or PDF

**Workflow 4: Cleaning Data**
1. Open Data Agent
2. Select mode (Normalize, Deduplicate, Clean Emails, Compare Lists)
3. Upload file(s)
4. Review changes
5. Download cleaned file

**Workflow 5: Getting Compliance Guidance**
1. Open Compliance Assistant
2. Select jurisdiction (Ontario)
3. Ask question about regulations
4. Receive cited, source-verified answer with disclaimers
5. Continue conversation or start new topic

**Workflow 6: Using Ask Lightspeed**
1. Open Ask Lightspeed
2. Type question or upload document (PDF/Excel/CSV)
3. AI uses tools autonomously (searches KB, calendar, Home Base, Shopify, etc.)
4. Confirm any write actions (create events, save to KB)
5. View response with inline charts if applicable

**Workflow 7: Team Communication (Home Base)**
1. Open Home Base
2. Create post with category (General, Urgent, FYI, Draw Update, Campaign)
3. Optionally attach files, schedule for later
4. Team members comment, react with emoji

**Workflow 8: Content Scheduling (Runway)**
1. Open Runway (Content Calendar)
2. Create event with title, date, category, recurrence
3. Add description and comments for team collaboration
4. Receive email reminders 24h before events

### 3.4 Role-Based Permissions

**Three Roles:**

| Role | Capabilities |
|------|-------------|
| **Owner** | Full access: manage members, roles, settings, billing, delete organization |
| **Admin** | Invite/remove members (except owners), edit org settings, manage KB |
| **Member** | Use all tools, create content, view team data (read-only for settings) |

**Additional:**
- **Super Admin** (system-level flag): Access global admin dashboard, platform-wide metrics, user/org management, bypass usage limits
- Feed URL fields (BUMP feeds) visible only to owner/admin, stripped for members
- Only owners can change member roles
- Compliance admin features restricted to super admins

### 3.5 Invitation & Team Onboarding

- Owners/admins send invitations via email
- Invitation expires in 7 days
- New invitees see onboarding wizard on first login if email matches
- Auto-accept if email matches pending invitation (no manual acceptance required)

### 3.6 Subscription & Billing

- 14-day free trial for new organizations (configurable via `TRIAL_DAYS` env var)
- Plans: Starter, Pro (monthly/annual via Stripe)
- Trial expiration triggers upgrade modal
- Subscription statuses: trial, active, past_due, cancelled
- Self-service cancellation via Stripe customer portal
- Usage limits: Trial (300 generations/month), Active (500/month), Past Due (50/month) — ⚠️ currently bypassed in code

## Part 4: Technical Architecture

### 4.3 Authentication

**Stateless JWT-based authentication with two OAuth providers.**

**Google OAuth 2.0:**
- Route: `POST /api/auth/google`
- Frontend: Google Identity Services (gsi/client)
- Backend: google-auth-library
- Flow: Frontend obtains ID token → sends to backend → backend verifies via `googleClient.verifyIdToken()` → extracts email/name/picture/google_id → creates user if new → auto-joins pending invitations → returns JWT

**Microsoft OAuth (Azure AD):**
- Route: `POST /api/auth/microsoft`
- Frontend: MSAL.js (msal-browser.min.js)
- Backend: @azure/msal-node (ConfidentialClientApplication)
- Authority: `https://login.microsoftonline.com/common`
- Scopes: `openid profile email User.Read`
- Flow: Frontend initiates popup → receives auth code → backend exchanges for tokens → fetches profile from Microsoft Graph API (`/me`) → creates user if new → returns JWT

**JWT Details:**
- Signed with `JWT_SECRET`
- Expiration: 7 days (configurable via `JWT_EXPIRES_IN`)
- Storage: `localStorage` key `authToken`
- No refresh tokens; users re-authenticate when expired
- No server-side session; logout is client-side token deletion

**Middleware Stack (`backend/src/middleware/auth.js`):**
- `authenticate()` — Verify JWT, load user from DB, cache org_id
- `requireOrganization()` — Load org from `:orgId` param, verify membership
- `requireAdmin` / `requireOwner` — Role-based access control
- `requireSuperAdmin` — System-level admin check (is_super_admin flag)
- `checkUsageLimit()` — Monthly usage quotas (⚠️ currently bypassed)
- `checkAIRateLimit()` — 10 AI requests/minute per user (in-memory Map tracking)

### 4.6 AI/LLM Integration

**Models Used:**
- **Claude Sonnet 4.6** (default): Main generation model (`process.env.ANTHROPIC_MODEL || 'claude-sonnet-4-6'`)
- **Claude Haiku 4.5**: Fast relevance picking for KB and examples (`process.env.HAIKU_MODEL || 'claude-haiku-4-5-20251001'`)

**API Configuration:**
- Endpoint: `https://api.anthropic.com/v1/messages`
- Headers: `anthropic-version: 2023-06-01`, `anthropic-beta: web-search-2025-03-05` (when web search tools present)

**Prompt Caching (Two-Layer Design):**
- **Layer 1 (Static):** Org-agnostic instructions (content types, writing principles, tone options, format rules). Cached via `cache_control: { type: 'ephemeral' }`. Reused across requests.
- **Layer 2 (Dynamic):** Per-request context: org name/mission/brand voice, email/social templates, upcoming calendar events, relevant KB entries, rated examples, brand voice fingerprint, response rules. Also cached with ephemeral control.

**Context Injection (per tool):**

| Context Layer | Response Asst. | Draft Asst. | Ask Lightspeed | Insights | List Normalizer |
|--------------|:-:|:-:|:-:|:-:|:-:|
| Knowledge Base | ✅ | ✅ | ✅ | light | ❌ |
| Response Rules | ✅ | ✅ | ✅ | ❌ | ❌ |
| Shopify Context | ✅ | ✅ | ✅ | ❌ | ❌ |
| Calendar Events | ✅ | ✅ | ✅ | light | ❌ |
| Conversation Memory | ✅ | ✅ | ✅ | ❌ | ❌ |
| Cross-Tool Context | ✅ | ✅ | ✅ | ✅ | ❌ |
| Voice Fingerprint | ✅ | ✅ | ✅ | ✅ | ❌ |

**KB Relevance Picking:**
- Uses Haiku to filter KB entries by relevance to inquiry
- Fallback: Tag-match scoring (tag substring +3, token overlap +1, title keyword +1)
- Returns top 8 most relevant entries

**Prompt Injection Protection:**
- Regex patterns detect: "ignore previous instructions", "disregard prior context", "you are now a", "reveal system prompt", etc.
- Detected patterns logged to audit and replaced with `[filtered]`
- Max inquiry length: 10,000 characters

**Output Validation:**
- Checks for HTML injection patterns
- Validates email formatting
- Detects format compliance (length, structure)
- Non-blocking (logs warnings but allows response)

**Agentic Tool Loop (Ask Lightspeed):**
1. User sends message (optional file upload)
2. Backend builds system prompt + tool definitions (12 tools + web_search)
3. Calls Claude with tools array
4. Claude responds with text + tool_use blocks
5. Read-only tools: execute immediately, add result to messages
6. Write tools: send confirmation dialog to frontend, await user confirm/deny
7. Loop back to Claude if tools called
8. Return final text response
